# Grill With Docs Command

Run the `grill-with-docs` skill now.

If `$ARGUMENTS` is non-empty, treat it as the initial plan context to interrogate.
If `$ARGUMENTS` is empty, ask for the plan first, then begin.

Interview one decision branch at a time, provide a recommended answer with each question, explore the codebase when it can answer the question, and update durable domain docs only when a real term or ADR-worthy decision crystallizes.
