# Migration from pie-player-components

This project (`@pie-players/pie-item-player`) is a ground-up rebuild of the item player originally provided by [`@pie-framework/pie-player-components`](https://github.com/pie-framework/pie-player-components). This document summarizes what changed and why.

## Unified element

The legacy project exposed two separate custom elements:

- `<pie-player>` for delivery
- `<pie-author>` for authoring

These are replaced by a single `<pie-item-player>` element with a `mode` attribute:

```html
<!-- Delivery -->
<pie-item-player mode="view" config="..." env="..." session="..."></pie-item-player>

<!-- Authoring -->
<pie-item-player mode="author" config="..." configuration="..."></pie-item-player>
```

## Loading strategies

The legacy player loaded IIFE bundles from the PIE build service exclusively, using script tag injection and `window.pie` for element registration.

The new player supports three strategies via the `strategy` attribute:

| Strategy | Description |
|----------|-------------|
| `iife` | IIFE bundles from a bundle host (same general approach as legacy, modernized) |
| `esm` | ESM modules from a CDN via import maps -- no script injection, tree-shakeable |
| `preloaded` | Host preloads element bundles at the page level; player skips all loading |

See [loading-strategies.md](./loading-strategies.md) for details.

## Modern stack

| Aspect | Legacy | Current |
|--------|--------|---------|
| Framework | Stencil 1.7 | Svelte 5 + Vite |
| TypeScript | 3.3 | 5.9 |
| Module format | CJS + IIFE (Stencil dist) | ESM-first (`"type": "module"`) |
| Registration | `defineCustomElements(window)` or auto via `<script>` | Auto-registers on import; `definePieItemPlayer()` for custom tag names |

## Modular loading primitive

The legacy `PieLoader` was a monolithic class that handled URL resolution, script injection, retry, and registry in a single file, and resolved its load promise opportunistically — sometimes before every requested tag was actually registered.

The new architecture replaces it with the `ElementLoader` primitive plus pluggable backends, all in `@pie-players/pie-players-shared`:

- `ensureRegistered(elements, { backend, ... })` / `assertRegistered(tags)` -- single primitive, truthful-promise contract enforced via a uniform post-load `customElements.whenDefined` verification pass
- IIFE backend (`loaders/iife-adapter.ts`) -- IIFE bundle loading and registration with bundle-build retry telemetry
- ESM backend (`loaders/esm-adapter.ts`) -- ESM dynamic-import loading via fully-qualified URLs or an injected import map
- `ItemController` -- Session state management (new; previously inlined in the player)
- `normalizeItemPlayerStrategy()`, `resolveItemPlayerView()` -- Strategy/view resolution utilities

## Tag naming

Legacy elements were registered with a `pp-` prefix (e.g. `pp-multiple-choice`).

The new player uses versioned tag names (e.g. `multiple-choice--version-9-9-1`) generated by `makeUniqueTags()`. This allows multiple versions of the same element to coexist on a page.

## Authoring media hooks

The legacy player relied on Stencil `@Method()` decorators and an implicit contract for media handling.

The new player exposes four explicit handler properties with two enforcement modes:

- `authoring-backend="demo"` -- built-in demo handlers (data URL based, non-production)
- `authoring-backend="required"` -- host must provide `onInsertImage`, `onDeleteImage`, `onInsertSound`, `onDeleteSound`; missing handlers block authoring and emit `player-error`

## Session management

The legacy player passed session data through to PIE elements with minimal normalization.

The new player uses `ItemController` to:

- Normalize sessions into `{ id: string, data: Array<{ id, element, ... }> }`
- Deduplicate `session-changed` events (metadata-only events are filtered out)
- Prevent response data from being overwritten by structural re-renders

## What is not changing

- **Bundle host URLs** -- `https://proxy.pie-api.com/bundles/` remains the default bundle host for IIFE strategy.
- **PIE element contract** -- Individual PIE elements (`@pie-element/*`) are unchanged. The player still renders them by setting `model`, `session`, and `env` properties on the registered custom elements.
- **Config shape** -- The `config` object structure (`{ elements, models, markup }`) is the same.
